# Git branching: `dev` и `main`

## Цель

Зафиксировать безопасный процесс разработки, чтобы не ломать прод-бота при доработках Mini App.

## Правила

1. Вся разработка ведется в ветке `dev`.
2. Ветка `main` используется только как продакшен-ветка.
3. Прямые рабочие изменения в `main` не вносятся.
4. Перенос из `dev` в `main` только через merge после проверки.
5. Перед merge обязательны:
   - автопроверки (`ruff`, `pytest`);
   - smoke-проверка текущего Telegram-бота;
   - smoke-проверка Mini App (если он включен).

## Деплой

1. Автодеплой на VPS запускается из `main` (workflow `.github/workflows/deploy-vps.yml`).
2. Из `dev` на VPS прод не деплоится автоматически.

## Для Mini App

1. Mini App развивается в `dev` как дополнительный канал.
2. Бот и Mini App должны работать параллельно на каждом этапе.
3. До релиза в `main` проверяются отсутствие регрессий bot-flow и корректность feature flags Mini App.
